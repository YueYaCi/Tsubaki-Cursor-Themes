# 投稿指南

感谢你愿意分享鼠标指针主题！请按以下步骤操作。

---

## 第一步：准备你的主题文件

新建一个文件夹，**用英文/数字命名**（建议用主题名拼音或英文名，如 `kafu-v1`）。

文件夹内必须包含：

```
你的主题文件夹名/
└── theme.json     ← 必须！主题的身份证
├── Arrow.cur      ← 至少包含一个正常选择光标
├── ...            ← 其他光标文件（可选）
└── preview.png    ← 预览图（可选，建议有）
```

**文件夹名不要用中文（如 可不），用英文 id（如 kafu-v1），因为下载 URL 里会包含这个路径。**

---

## 第二步：填写 theme.json

复制下面的模板，修改后放在你的主题文件夹里：

```json
{
  "name": "主题名称（显示在软件里的名字）",
  "author": "作者名或社团名",
  "version": "1.0",
  "description": "一句话介绍",
  "source": "原出处链接（如果是原创，可写你的主页）",
  "license": "同人非商用 / 原创 / CC0 / 禁止转载 / 等等"
}
```

字段说明：

| 字段 | 必填 | 说明 |
|------|------|------|
| name | ✅ | 显示在软件里的主题名称 |
| author | ✅ | 作者名，原创写你自己，搬运写原作者 |
| version | ✅ | 版本号，如 1.0 |
| description | ❌ | 简介，一句话即可 |
| source | ✅ | 原出处 URL，搬运必须填写 |
| license | ✅ | 授权说明，建议写"原创""同人非商用""禁止转载"等 |

---

## 第三步：检查文件命名

你的光标文件必须能被软件识别。支持以下命名（不区分大小写）：

| 系统角色 | 支持的文件名 |
|----------|--------------|
| 正常选择 | Arrow / Normal / Pointer |
| 帮助 | Help |
| 后台运行 | AppStarting / Working |
| 忙 | Wait / Busy |
| 文本选择 | IBeam / Text |
| 手写 | NWPen / Handwriting |
| 不可用 | No / Unavailable |
| 垂直调整 | SizeNS / Vertical / Vert |
| 水平调整 | SizeWE / Horizontal / Horz |
| 对角线调整1 | SizeNWSE / Diagonal2 / Dgn2 |
| 对角线调整2 | SizeNESW / Diagonal1 / Dgn1 |
| 移动 | SizeAll / Move |
| 候选 | UpArrow / Alternate |
| 链接选择 | Hand / Link |
| 定位 | Pin / Loc |
| 个人 | Person |

文件名不需要包含全部 17 个，缺少的角色会保持用户原有设置。

---

## 第四步：提交投稿

**方式 A：通过 GitHub Pull Request（推荐，会 Git 的用户）**

- Fork 本仓库
- 把你的主题文件夹放进 `themes/`
- 提交 PR，标题写：`添加主题：主题名`
- PR 描述里必须写清楚：
  - 你是原创作者还是搬运？
  - 如果是搬运，原作者是否允许分发？
  - 来源链接是什么？

**方式 B：通过 GitHub Issue（不会 Git 的用户）**

- 发一个新 Issue，标题写：`[投稿] 主题名`
- 把 `theme.json` 的内容贴在正文里
- 把光标文件打包成 `.zip` 上传到 Issue 附件
- 维护者会帮你整理进仓库

---

## 审核标准

维护者会检查以下内容，通过后才合并：

- [ ] theme.json 填写完整
- [ ] 文件命名符合规范（至少包含一个可识别的光标）
- [ ] 如果是搬运，来源清晰且无明显侵权风险
- [ ] 不包含恶意文件（.exe、.bat 等会被拒绝）

---

## 提醒

- 原创主题：投稿即视为同意在本仓库展示，版权归你所有。
- 搬运主题：请确保你有权分享，或原作者允许转载。若后续收到侵权投诉，该主题将被移除。
- 禁止投稿：付费资源、加密资源、明确标注"禁止转载"的资源。
